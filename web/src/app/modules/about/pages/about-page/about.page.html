About

Welcome to HelloTangle!

This is a web application leveraging IOTA, a truly promising ecosystem of technologies that utilizes the IOTA Foundation's (IF) implementation of a distributed ledger technology (DLT) known as the Tangle. HelloTangle acts as a simple bridge between a user and the IOTA network, allowing one the ability to quickly and easily send messages to any given address. The frontend (what you're using now) is built with Angular, while the backend (where the actual message broadcasting takes place) was built with NestJS. To see more of the technical details, checkout the repo!

To give some explanation about IOTA, it is a new internet protocol that is intended for transacting between machines that are a part of the IoT (Internet of Things). Many people know it as a cryptocurrency, which to be fair it was inspired mainly by that realm of technologies (i.e. Bitcoin, Ethereum), but it aims to be much more than that in a few ways:

  • Implementing the Tangle to store transactions as opposed to a blockchain data structure like what Bitcoin or Ethereum use (a.k.a linked lists). Because the Tangle is essentially a directed acyclic graph (DAG), the network is not bottlenecked by miners needing to confirm transactions like in Bitcoin, meaning that it offers much faster confirmation times. This means that IOTA transactions are feeless since you (or your devices) are the entities responsible for confirming transactions so that yours will be confirmed.
  • Offering data-only transactions as well as microtransactions, which affords the m2m (machine-to-machine) economy a much more efficient and secure way of transferring value or data between devices. This also carries huge implications for how our data with be handled in the future as there are projects being developed, namely Alvarium, that are establishing methods of auditing data for authenticity, confidence, and so forth.
  • Aiming to be the primary network protocol used by the m2m economy while offering engineers a vast ecosystem of libraries and tools for interacting with the IOTA network. Those could be libraries for running nodes to process transactions, backend servers creating transactions, or handling sensitive and confidential data.
